O Problema

RAG é Cego ao Tempo — 4 Dimensões

Cada paper ataca uma dimensão distinta da temporalidade — quatro problemas ortogonais

Temporal Blur

CEO da empresa X em 2021 e CEO em 2024 têm embeddings similares — o retriever não distingue qual versão é relevante para "agora".

Custo de Token

GraphRAG padrão busca todos os eventos de todos os anos — semanticamente relevantes mas temporalmente ruidosos, inflando o contexto.

Version Conflation

Docs de versões diferentes (v14, v15, v16) misturados — a resposta correta depende de qual versão o usuário está usando.

Knowledge Drift

LLM com cutoff de 2023 não sabe o que mudou em 2024-2025 — fino-tuning contínuo causa catastrophic forgetting.

Metáfora · TG-RAG (HKUST + U Turku + HSBC, Out 2025)
🗃️
O Arquivo Policial com Linha do Tempo

Um arquivo policial eficiente organiza crimes em dois níveis: os fatos concretos (quem, o quê, quando) e uma hierarquia de tempo (por ano, trimestre, mês). O detetive primeiro responde "de quando estamos falando?" antes de buscar os fatos. Ele não confunde o depoimento de 2020 com o de 2024 sobre o mesmo suspeito — cada fato carrega seu carimbo temporal como quádruplo (entidade, relação, valor, timestamp). TG-RAG é esse arquivo policial: dois grafos distintos, dois papéis distintos.

Quádruplo Temporal
(v₁, v₂, relação, τ) — ex: (EmpA, CEO, X, 2021-Q1). Fatos da mesma entidade em datas diferentes ficam como arestas paralelas distintas — sem blur semântico.
Hierarquia de Tempo
Ano→Trimestre→Mês→Dia. Summaries multi-granulares propagados de baixo para cima. Visão abrangente (tendências) ou detalhada (fatos precisos) conforme a query.
TG-RAG · HKUST + U Turku + HSBC · Out 2025

Dois Grafos, Dois Papéis

Camada temporal (hierarquia ano→trimestre) + camada de fatos (quádruplos). ECT-QA: 480 earnings calls, 24 empresas, 2020-2024, 1,58M tokens

Recuperação em 3 estágios: (1) LLM identifica escopo temporal da query → (2) cosine similarity filtrado por timestamp → subgrafo V_t^q → (3) PPR local (fatos precisos) + summaries globais dos nós temporais (tendências). Atualização incremental: novos docs inserem apenas em nós folha, propagação sobe apenas nos ancestrais afetados.

Animação Conceitual · Temporal Window Selection

O Problema do Blur Temporal — Resolvido

Sem roteamento temporal, todos os fatos da entidade recebem o mesmo score. Com janela temporal ativa, apenas os fatos do período correto são recuperados

Sem Roteamento Temporal

Query "CEO da Emp X" → todos os anos ativados → CEO=A (2020), CEO=B (2022), CEO=C (2024) misturados → LLM responde errado

Com Janela Temporal (TG-RAG)

Query "CEO da Emp X em 2024" → identifica τ=2024 → apenas CEO=C ativo → resposta correta e rastreável

Metáfora · STAR-RAG (NTU, Out 2025, ICLR 2026)
🌦️
O Meteorologista e os Padrões Climáticos

Um meteorologista não analisa cada observação climática individual para fazer a previsão. Ele trabalha com categorias de padrões — "frente fria que se segue a alta pressão" — e foca nos eventos do intervalo correto. O GraphRAG padrão equivale ao meteorologista que lê todas as observações de todos os anos sem filtro. STAR-RAG é o meteorologista que comprime em padrões via MDL e filtra pelo intervalo correto — 97% menos tokens, 9,1% mais preciso, sem treinar nada.

1 · Entity Labeling

Apriori extrai subconjuntos frequentes → label estrutural por entidade

2 · Rule Nodes

(s,r,o,t) → ⟨c_s, r, c_o⟩ — evento individual vira padrão reutilizável

3 · MDL + PPR

Minimiza L(G|M) = coverage_cost + temporal_cost → PPR alinhado ao tempo

STAR-RAG · NTU · Out 2025 · ICLR 2026

+9,1% Accuracy com 97% Menos Tokens

Baseline: MedicalGraphRAG = 35,3% accuracy, 0,97 token budget. STAR-RAG = 44,4%, 0,03 token budget

Por que funciona: GraphRAG recupera eventos temporalmente dispersos (semanticamente similares mas de anos diferentes). STAR-RAG restringe a propagação ao subgrafo temporal alinhado à query — menos noise, menos tokens no contexto, melhor geração. Zero fine-tuning necessário.

Metáfora · VersionRAG (ZHAW, Suíça, Out 2025)
📦
O Dev Perdido na Documentação

Um desenvolvedor pergunta: "Qual é o nível de estabilidade de assert.deepEqual() na versão 15.14.0 do Node.js?" O RAG padrão retorna trechos da v14.21.3 ("experimental"), v15.14.0 ("stable") e v16.20.2 ("deprecated") misturados sem distinção. O LLM vê os três e escolhe errado. Pior: quando o dev pergunta "em qual versão assert.deepEqual() foi removida?", o RAG nem entende que a pergunta é sobre uma mudança — não sobre um conteúdo. VersionRAG usa um grafo de 5 níveis para separar os dois mundos.

VersionRAG · ZHAW · Out 2025

5 Níveis, 3 Caminhos de Recuperação

VersionQA: 100 Q&A manuais em 34 docs técnicos (Apache Spark, Bootstrap, Node.js Assert, Node.js Errors) · 186K tokens de indexação vs 2.970K do GraphRAG

Change Detection Implícita

60% accuracy em mudanças não documentadas (diffs implícitos). GraphRAG: 10%. Naive RAG: 0%. DeepDiff library + LLM gera descrição semântica da mudança automaticamente.

Eficiência de Indexação

186K tokens (VersionRAG) vs 2.970K (GraphRAG) = 97% menos tokens. Versões são arestas do grafo — não extraídas por LLM.

Metáfora · Chronos (Tsinghua + UIUC + CUHK, Abr 2026)
🗞️
O Jornalista com Memória Congelada

Um jornalista que só leu jornais até 2023 pergunta sobre "o homem mais rico do mundo em agosto de 2025". Sua memória diz "Elon Musk" — mas o mundo mudou. Drift contínuo significa que entidades evoluem: CEOs mudam, políticas mudam, preços de ações flutuam. O problema não é só ter a informação nova — é raciocinar de forma temporalmente consistente com múltiplos eventos em sequência ("em Jan-2024 era X, em Jun-2024 passou a ser Y, então..."). Fine-tuning contínuo resolve parcialmente mas causa catastrophic forgetting. Chronos usa um Event Evolution Graph para conectar os eventos sem retreinar nada.

EEG em 3 Estágios para Consistência Temporal

Tensão Conceitual · O Debate

Para Grafos Temporais:
Riqueza Estrutural ou Eficiência de Token?

🗃️ TG-RAG: Riqueza Estrutural

Tese: o grafo bi-nível com summaries hierárquicos garante tanto precisão factual (via quádruplos) quanto visão de tendência (via summaries propagados). Adequado para corpora densos (1,58M tokens, 24 empresas, 5 anos de earnings calls).

ECT-QA benchmark único para temporal KG corporativo · PPR local + summaries globais
🌦️ STAR-RAG: Eficiência de Token

Tese: comprimir eventos em rule nodes via MDL elimina 97% do custo de token sem perder precisão — ao contrário, ganha +9,1% de accuracy porque o ruído temporal diminui. Funciona sem fine-tuning. Adequado para deployment em produção com budget de contexto restrito.

35,3%→44,4% accuracy · 0,97→0,03 token budget

Síntese dependente de contexto: TG-RAG é superior quando o corpus é denso e rico em fatos temporais (earnings calls, relatórios anuais, histórico corporativo) e quando summaries de tendência são necessários. STAR-RAG é superior quando o budget de token é restrito, quando o corpus é grande e temporalmente ruidoso, ou quando zero fine-tuning é um requisito. Para ambientes de produção — STAR-RAG. Para análise profunda de corpora corporativos — TG-RAG.

Módulo 08 · Princípios Destilados

A Biblioteca Finalmente Lembra o Calendário

Lembram dos 4 especialistas? A biblioteca que esquecia o calendário (intro) aprendeu quatro lições distintas. O arquivo policial (TG-RAG) ganhou sua linha do tempo bi-nível — cada quádruplo tem seu carimbo e os summaries sobem hierarquicamente. O meteorologista (STAR-RAG) comprimiu 97% dos registros meteorológicos sem perder os padrões — e ficou 9,1% mais preciso, não menos. O dev perdido (VersionRAG) ganhou o mapa de 5 níveis — agora ele sabe se a pergunta é sobre conteúdo, sobre versão, ou sobre uma mudança. E o jornalista (Chronos) aprendeu a raciocinar em sequência temporal — "em Jan-2024 era X, em Jun-2024 passou a ser Y, portanto em Ago-2025 é Z" — sem retreinar a memória, apenas conectando os eventos. O RAG temporal não é uma extensão — é um paradigma diferente de relação com o conhecimento.

P1

Timestamps são cidadãos de primeira classe — (entidade, relação, valor, τ) como quádruplo; nunca misture fatos de épocas diferentes no mesmo embedding.

P2

Menos tokens pode ser mais accuracy — STAR-RAG comprimir 97% do ruído temporal resultou em +9,1%. Não assuma que mais contexto é sempre melhor.

P3

Separe conteúdo de mudança — VersionRAG tem nós dedicados para mudanças explícitas e implícitas. Uma query sobre mudança não é uma query sobre conteúdo.

P4

Fine-tuning contínuo é armadilha — catastrophic forgetting destrói o histórico. EEG e grafos temporais recuperam consistência sem retreinar o modelo base.
