Por que a query original falha?
O gargalo silencioso que limita todo sistema RAG
Ruído e Intenção Desviada

Usuários raramente formulam perguntas ideais para recuperação. Incluem contexto irrelevante, omitem termos-chave ou expressam ambiguidade de intenção.

Falta de Diversidade

Uma única query reescrita ainda captura apenas uma perspectiva do espaço de documentos relevantes, perdendo evidências complementares.

Queries Complexas

48,3% das queries reais requerem decomposição; 37,6% têm múltiplas intenções. Abordagens de query única falham sistematicamente nesses casos.

Custo de Retreinamento

Métodos supervisionados exigem anotações caras (78h/30k instâncias). Métodos de prompt não generalizam. Há uma lacuna de feedback eficiente e barato.

"Queries com ruído e desvio de intenção necessitam de rewriting para melhorar a relevância dos documentos recuperados." — DMQR-RAG, Li et al., Kuaishou Technology / Renmin University, Nov 2024

DMQR-RAG: O Advogado das Quatro Perguntas
Diverse Multi-Query Rewriting — Renmin University + Kuaishou, Nov 2024
Metáfora
Um advogado experiente nunca depende de uma única pergunta ao testemunhar. Ele reformula o mesmo fato de quatro ângulos: pergunta geral para limpar o ruído, extrai apenas as palavras-chave, cria uma resposta hipotética para orientar a busca e, por fim, elimina os excessos para chegar ao núcleo. Cada versão captura evidências diferentes — juntas, cobrem o caso inteiro.
Query original = cliente expressando mal o problema · GQR = pergunta limpa e geral · KWR = só os termos essenciais · PAR = hipótese de resposta como isca · CCE = núcleo sem ruído
  • 1
    GQR (General Query Rewriting): remove ruído, preserva toda a informação relevante — aumenta precisão de recuperação
  • 2
    KWR (Keyword Rewriting): extrai substantivos e sujeitos — ideal para motores de busca por palavra-chave
  • 3
    PAR (Pseudo-Answer Rewriting): gera resposta hipotética como documento-âncora — captura semântica da resposta esperada
  • 4
    CCE (Core Content Extraction): queries longas demais → descarta detalhes, extrai essência — reduz ruído no reranker

RaFe: O Professor que Aprende com as Notas
Ranking Feedback Improves Query Rewriting — EMNLP 2024, Zhejiang / Alibaba
Metáfora
Um professor que melhora as questões do exame não precisa de um gabarito externo — ele usa as próprias notas dos alunos. Se a maioria errou, a questão estava mal formulada; se todos acertaram facilmente, foi simples demais. O rankeamento dos documentos recuperados funciona como essas notas: revela imediatamente se a query reescrita foi boa ou ruim — sem precisar de anotadores humanos.
Query reescrita = questão do exame · Documentos recuperados = respostas dos alunos · Score do reranker = nota automática · DPO/KTO/PPO = revisão pedagógica do professor
Abordagens anteriores
✗ Precisam de documentos anotados
✗ Dependem de respostas coretas como reward
✗ Custo: 78h para 30k instâncias com LLM
✗ Não generalizam entre domínios
RaFe
✓ Reranker público como feedback (bge-reranker)
✓ Sem anotações — S(q,q') = média dos scores
✓ Custo: apenas 0,67h para 30k instâncias
✓ Cross-lingual: EN + ZH sem mudança

Query Decomposition: O Minerador das Muitas Galerias
Exploration-Exploitation for RAG — Amsterdam + Johns Hopkins, Out 2025
Metáfora
Um minerador inteligente não escava uma galeria até o fim antes de explorar as outras. Ele sonda cada túnel um pouco, memoriza o que encontrou e concentra esforço nas galerias mais promissoras — abandonando cedo as estéreis. O orçamento de extração é fixo; a arte está em alocar melhor onde cavar. Cada sub-query é uma galeria; cada documento, uma pedra avaliada.
Query complexa = mina com múltiplas galerias · Sub-queries = braços do bandido (MAB) · Documento recuperado = pedra avaliada · Bernoulli top-k UCB = estratégia de alocação de esforço
  • →
    Exploração: todo sub-query explorado pelo menos uma vez (UCB: c·√(log₂(n+1)/n))
  • →
    Explotação: sub-queries com alto Beta posterior recebem mais documentos
  • →
    Diversidade: novelty = 1 − max_cos(d_novo, observados_anteriores) / 2
  • →
    Hierarquia: sub-queries promissores geram filhos (herança Beta: λα, λβ)

ACQO: O General que Adapta a Estratégia ao Campo
Adaptive Complex Query Optimization with RL — Tencent Youtu / HKU / NTU, Jan 2026
Metáfora
Um general sábio não aplica a mesma tática em todo terreno. Em campo aberto, divide as forças (decompõe a query em sub-queries); em território ambíguo, manda batedores para clarificar (disambiguação); em terreno conhecido, ataca direto. E ao fundir os relatórios dos batalhões, não confia só na hierarquia de patentes — também considera a força absoluta da informação trazida.
Query complexa = campo de batalha · AQR = decisão tática (dividir/disambiguar/expandir) · RSF = fusão de relatórios por rank E score · CRL duas etapas = treino progressivo: recrutas → casos difíceis

A Tensão Central: Diversidade vs. Precisão
Mais queries = mais recall; mas também mais ruído e custo computacional
Argumento da Diversidade

Queries diversas cobrem diferentes partes do espaço semântico. DMQR-RAG mostra +10% P@5 sobre RAG-Fusion que usa queries similares. Documentos relevantes distintos emergem de ângulos distintos.

Problema do Ruído

Mais rewrites introduzem documentos irrelevantes. RaFe mostra que "bad rewrites" têm Prec@5=30,74 — pior que OQR (41,41). O reranker é essencial para filtrar após expansão.

Seleção Adaptativa

DMQR-RAG e ACQO convergem para a mesma solução: não aplicar todas as estratégias a toda query. Seleção por few-shot (DMQR) ou RL (ACQO) escolhe o mínimo eficaz.

Orçamento como Design

O framework bandit formaliza isso: budget fixo b distribuído dinamicamente entre sub-queries. ε-greedy performa melhor em b=10–20% — exploração precoce de braços de alto valor é ótima.

"Aumentar o número de rewrites não é sempre benéfico — pode introduzir ruído. A seleção adaptativa de estratégias elimina rewrites desnecessários sem comprometer a recuperação relevante." — DMQR-RAG, Seção 3.2.2

Mapa de Decisão: Qual Estratégia Usar?
Guia prático baseado nos achados experimentais dos quatro artigos
  • 🔀
    DMQR-RAG — quando as queries têm ruído variável e você precisa de um framework plug-and-play com LLMs existentes. Ideal para domínio aberto com motor de busca web.
  • 🔄
    RaFe — quando quer treinar um modelo de rewriting pequeno (7B) sem anotações caras. Melhor ganho no cenário EXPAND-Ranked: usa original + rewrite, depois re-ranqueia tudo.
  • ⛏️
    Bandit Decomposition — para queries longas e factuais que se decompõem naturalmente em sub-tópicos independentes. Budget limitado? Use ε-greedy a 10–20%. Respostas longas? Use Bernoulli k=4 para cobrir nugets.
  • 🎖️
    ACQO — para sistemas conversacionais (multi-turn) e multi-hop onde a decisão de decompor ou não deve ser aprendida. Ambientes com retrievers heterogêneos (sparse + dense): RSF é mais robusto que RRF.
Query única + ruído → DMQR Treinar rewriter → RaFe Multi-sub-query + budget → Bandit Conversacional + RL → ACQO

O Que os Quatro Artigos Concordam
Achados convergentes que definem o estado da arte em query rewriting
🏆
Reranker é indispensável. RaFe, DMQR e ACQO mostram que reranking após expansão amplifica os ganhos — sem ele, a expansão pode ser neutra ou prejudicial.
📐
Query original tem valor. DMQR e RaFe incluem q original no pool. Rewrites que desviam demais da intenção perdem precisão — OQR supera métodos ruins.
🌐
Generalização sem fine-tuning. ACQO generaliza zero-shot para MultiHop-RAG; DMQR funciona com Llama3, Qwen2 e GPT-4; RaFe transfere EN→ZH sem retraining.
📉
Mais ≠ Melhor. Bandit mostra que budget b=10–20% pode igualar b=100% com políticas inteligentes. DMQR mostra que seleção reduz rewrites sem perda de qualidade.
🔗
Diversidade semântica importa. O ganho de DMQR sobre RAG-Fusion vem da diversidade informacional, não da quantidade. Queries similares não somam.
⚙️
Seleção adaptativa é o caminho. Todo método converge para seleção contextual: few-shot (DMQR), feedback (RaFe), bandit (Petcu), RL+CRL (ACQO).

Princípios do Query Rewriting Moderno
Destilados dos quatro artigos — aplicáveis em qualquer pipeline RAG
01
Diversidade informacional, não numérica
Cada query reescrita deve trazer informação única que as outras não têm. GQR, KWR, PAR e CCE cobrem quatro dimensões ortogonais. Queries semelhantes não somam cobertura.
02
Feedback alinhado com o objetivo de recuperação
O reranker é o sinal de feedback mais eficiente: custo 100× menor que LLM-judge, sem necessidade de anotações, e alinhado diretamente com o objetivo de retrieval. RaFe prova isso.
03
Orçamento de recuperação como variável de design
Tratar o número de documentos como recurso finito a alocar (não parâmetro fixo) permite ganhos de 35% em precisão sem aumentar custo total. O framework bandit formaliza isso.
04
Complexidade da query determina estratégia
Queries simples: single rewrite com GQR. Queries ambíguas: decomposição + bandit. Queries conversacionais multi-hop: RL adaptativo (ACQO). Não existe estratégia universal.
05
Fusão por rank E score absoluto supera RRF puro
RSF do ACQO mostra que ignorar scores absolutos (como RRF faz) perde informação em empates. P(p) prioriza consenso de rank; S(p) desempata pela evidência mais forte.
